# 2. 핵심 게임 로직 설계

## 편지의 두 가지 종류

### 일반 편지
- 수신자 미지정, 태그만 설정
- 작성 완료 시 서버가 즉시 첫 번째 경유지 계산
- 각 경유지에서 홉별로 다음 경유지 계산 (태그 가중치 기반)
- 태그 일치율 100% 우체국 도달 시 수신 선택지 제공
- 수신자가 선택하면 전달 완료

### 중앙 우체국 편지
- 수신자 직접 지정
- 중앙 우체국을 통해 직통 전달
- 월 기본 3회, 재화로 최대 2회 추가 (월 최대 5회)

## 경유자 정보 제한
각 경유자는 아래만 알 수 있다:
- `prev_holder`: 직전 우체국
- `next_holder`: 다음 전달할 우체국
- `tags`: 편지 태그

송신자, 수신자, 전체 경로, 편지 내용은 알 수 없다.

## 방문 및 대화 규칙
- 관계 없는 우체국: 방문 불가
- 관계 있는 우체국 + 전달할 편지 없음: 구경, 흔적 남기기만
- 관계 있는 우체국 + 전달할 편지 있음: 구경 + 짧은 대화 (방문당 최대 5번)
- 대화는 기록되지 않음, 해당 방문 순간에만 존재

### letters
```sql
CREATE TABLE letters (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sender_id        UUID REFERENCES users(id),
  letter_type      TEXT NOT NULL,
                       -- 'normal'  : 일반 편지 (태그 기반 라우팅)
                       -- 'central' : 중앙 우체국 편지 (직통)
  tags             TEXT[] DEFAULT '{}',       -- 편지 태그
  content          TEXT NOT NULL,             -- 봉인된 내용 (경유자 접근 불가)
  stamp_type       TEXT,
  status           TEXT DEFAULT 'waiting',
                       -- waiting   : 현재 경유지 대기 중
                       -- delivered : 수신자에게 도달
  current_holder   UUID REFERENCES users(id), -- 현재 편지 보유자
  prev_holder      UUID REFERENCES users(id), -- 직전 경유자 (경유자에게만 노출)
  next_holder      UUID REFERENCES users(id), -- 다음 경유지 (서버 계산, 경유자에게만 노출)
  recipient_id     UUID REFERENCES users(id), -- 중앙우체국 편지만 사용
  visited_users    UUID[] DEFAULT '{}',       -- 순환 방지용 경유 기록
  created_at       TIMESTAMPTZ DEFAULT now(),
  delivered_at     TIMESTAMPTZ
);
```

> `content`는 RLS로 `current_holder`(경유자)에게도 접근 불가.
> 오직 `sender_id`와 최종 `recipient`만 열람 가능.

# 4. 핵심 시스템 구현

## 4-1. 홉별 라우팅 — 다음 경유지 계산 (Edge Function)

편지 작성 시 최초 1회, 이후 각 전달 시마다 실행.

```typescript
// supabase/functions/calculate-next-hop/index.ts
serve(async (req) => {
  const { letter_id, current_holder_id } = await req.json();
  const supabase = createClient(...);

  // 1. 편지 태그 조회
  const { data: letter } = await supabase
    .from('letters')
    .select('tags, visited_users')
    .eq('id', letter_id)
    .single();

  // 2. 현재 보유자의 관계망 조회
  const { data: connections } = await supabase
    .from('relationships')
    .select('to_user_id')
    .eq('from_user_id', current_holder_id);

  // 3. 이미 거친 우체국 제외
  const candidates = connections
    .map(c => c.to_user_id)
    .filter(id => !letter.visited_users.includes(id));

  // 4. 후보 우체국의 관심 태그 조회
  const { data: offices } = await supabase
    .from('post_offices')
    .select('user_id, interest_tags')
    .in('user_id', candidates);

  // 5. 태그 일치율 계산
  const scored = offices.map(office => {
    const matches = office.interest_tags.filter(
      tag => letter.tags.includes(tag)
    ).length;
    const score = matches / letter.tags.length; // 0.0 ~ 1.0
    return { user_id: office.user_id, score, matches };
  });

  // 6. 일치율 100% 우체국 존재 시 → 수신 선택지 제공
  const perfectMatch = scored.find(s => s.score === 1.0);
  if (perfectMatch) {
    await supabase.from('letters').update({
      next_holder: perfectMatch.user_id,
      status: 'pending_acceptance'  // 수신 대기
    }).eq('id', letter_id);

    // 수신 후보자에게 FCM 알림
    await sendPushNotification(
      perfectMatch.user_id,
      '당신에게 닿은 편지가 있어요'
    );
    return new Response(JSON.stringify({ type: 'match_found' }));
  }

  // 7. 일치율 기반 정렬 → 상위 후보 중 다음 홉 선택
  scored.sort((a, b) => b.score - a.score);
  const nextHop = scored[0]; // 가장 높은 일치율

  await supabase.from('letters').update({
    next_holder: nextHop.user_id,
    prev_holder: current_holder_id,
  }).eq('id', letter_id);

  return new Response(JSON.stringify({
    type: 'next_hop',
    next_holder: nextHop.user_id
  }));
});
```

## 4-2. 편지 직접 전달 (Edge Function)

유저가 다음 우체국을 방문해 "전달하기"를 누를 때 실행.

```typescript
// supabase/functions/relay-letter/index.ts
serve(async (req) => {
  const { letter_id, actor_user_id } = await req.json();
  const supabase = createClient(...);

  const { data: letter } = await supabase
    .from('letters')
    .select('*')
    .eq('id', letter_id)
    .single();

  // 1. 현재 보유자 검증
  if (letter.current_holder !== actor_user_id) {
    return new Response('Unauthorized', { status: 403 });
  }

  // 2. 편지 상태 업데이트
  await supabase.from('letters').update({
    current_holder: letter.next_holder,
    prev_holder:    actor_user_id,
    next_holder:    null,  // 다음 홉은 새로 계산
    visited_users:  [...letter.visited_users, actor_user_id],
  }).eq('id', letter_id);

  // 3. 경유 기록
  await supabase.from('letter_logs').insert({
    letter_id,
    from_user_id: actor_user_id,
    to_user_id:   letter.next_holder,
  });

  // 4. 경유 보상
  await rewardRelay(supabase, actor_user_id);

  // 5. 다음 홉 계산 (새 경유자 기준으로)
  await calculateNextHop(letter_id, letter.next_holder);

  // 6. 새 보유자에게 알림
  await sendPushNotification(
    letter.next_holder,
    '당신의 우체국에 편지가 왔어요'
  );

  return new Response('ok');
});
```

## 4-3. 수신 선택 처리 (Edge Function)

태그 일치 유저가 "수신하기"를 누를 때.

```typescript
// supabase/functions/accept-letter/index.ts
serve(async (req) => {
  const { letter_id, acceptor_id } = await req.json();
  const supabase = createClient(...);

  // 1. 편지 수신 완료 처리
  await supabase.from('letters').update({
    status:        'delivered',
    current_holder: acceptor_id,
    recipient_id:  acceptor_id,
    delivered_at:  new Date().toISOString(),
  }).eq('id', letter_id);

  // 2. 편지 교환 횟수 업데이트 → 관계 형성 판정
  const { data: letter } = await supabase
    .from('letters').select('sender_id').eq('id', letter_id).single();

  await updateExchangeAndFormRelationship(
    supabase, letter.sender_id, acceptor_id
  );

  // 3. 송신자에게 알림 (편지가 누군가에게 닿았다는 사실만 알림)
  await sendPushNotification(
    letter.sender_id,
    '흘려보낸 편지가 누군가에게 닿았어요'
  );

  return new Response('ok');
});
```

## 4-4. 관계 형성 자동 판정

```sql
CREATE OR REPLACE FUNCTION update_exchange_and_form_relationship(
  p_sender_id    UUID,
  p_recipient_id UUID
) RETURNS VOID AS $$
DECLARE
  v_user_a UUID;
  v_user_b UUID;
  v_count  INTEGER;
BEGIN
  v_user_a := LEAST(p_sender_id, p_recipient_id);
  v_user_b := GREATEST(p_sender_id, p_recipient_id);

  INSERT INTO letter_exchanges (user_a_id, user_b_id, exchange_count)
  VALUES (v_user_a, v_user_b, 1)
  ON CONFLICT (user_a_id, user_b_id)
  DO UPDATE SET
    exchange_count = letter_exchanges.exchange_count + 1,
    updated_at = now()
  RETURNING exchange_count INTO v_count;

  -- 3회 이상이면 양방향 관계 생성
  IF v_count >= 3 THEN
    INSERT INTO relationships (from_user_id, to_user_id)
    VALUES (v_user_a, v_user_b)
    ON CONFLICT DO NOTHING;

    INSERT INTO relationships (from_user_id, to_user_id)
    VALUES (v_user_b, v_user_a)
    ON CONFLICT DO NOTHING;
  END IF;
END;
$$ LANGUAGE plpgsql;
```

```typescript
async function createVisitChat(letter_id, visitor_id, host_id) {

// 메시지 전송 (최대 5번 제한)
async function sendChatMessage(chat_id, sender_id, content) {
  const { data: chat } = await supabase
    .from('visit_chats')
    .select('message_count')
    .eq('id', chat_id)
    .single();

  if (chat.message_count >= 5) {
    return new Response('Message limit reached', { status: 429 });
  }

  await supabase.from('visit_chat_messages').insert({
    chat_id, sender_id, content
  });

  await supabase.from('visit_chats').update({
    message_count: chat.message_count + 1
  }).eq('id', chat_id);
}

}
```

```typescript
  // 관계 수가 적은 유저 2~3명 조회 (관계 적은 순 정렬)

      { from_user_id: new_user_id, to_user_id: candidate.id },
      { from_user_id: candidate.id, to_user_id: new_user_id },
    ]);
  }

  return new Response('ok');
});
```

## 4-8. 중앙 우체국 (월 최대 5회)

```typescript
// supabase/functions/register-to-central/index.ts
const MONTHLY_BASE_LIMIT = 3;
const MONTHLY_MAX_LIMIT  = 5;  // 재화로 추가 최대 2회

serve(async (req) => {
  const { letter_id, user_id, use_extra } = await req.json();
  const supabase = createClient(...);

  const yearMonth = new Date().toISOString().slice(0, 7);

  const { data: usage } = await supabase
    .from('central_post_office_usage')
    .select('used_count, extra_count')
    .eq('user_id', user_id)
    .eq('year_month', yearMonth)
    .maybeSingle();

  const usedCount  = usage?.used_count  ?? 0;
  const extraCount = usage?.extra_count ?? 0;
  const limit = MONTHLY_BASE_LIMIT + extraCount;

  if (usedCount >= limit) {
    if (extraCount >= 2) {
      return new Response(
        JSON.stringify({ error: 'monthly_max_reached' }), { status: 429 }
      );
    }
    if (!use_extra) {
      return new Response(
        JSON.stringify({ error: 'base_limit_exceeded', can_use_extra: true }),
        { status: 429 }
      );
    }
    // 재화 소모 후 추가 이용
    await deductCurrency(supabase, user_id);
    await supabase.from('central_post_office_usage').upsert({
      user_id, year_month: yearMonth,
      extra_count: extraCount + 1
    }, { onConflict: 'user_id, year_month' });
  }

  // 편지 등록
  await supabase.from('central_post_office_letters').insert({
    letter_id, registered_by: user_id
  });

  await supabase.from('central_post_office_usage').upsert({
    user_id, year_month: yearMonth,
    used_count: usedCount + 1
  }, { onConflict: 'user_id, year_month' });

  return new Response(JSON.stringify({
    ok: true,
    remaining: limit - usedCount - 1
  }));
});
```

```sql
BEGIN
  IF p_strength = 10 THEN
    INSERT INTO unlocks (user_a_id, user_b_id, unlock_type)
    VALUES (p_user_a, p_user_b, 'joint_stamp') ON CONFLICT DO NOTHING;
  END IF;

    INSERT INTO unlocks (user_a_id, user_b_id, unlock_type)
    VALUES (p_user_a, p_user_b, 'secret_music') ON CONFLICT DO NOTHING;
  END IF;
  IF p_strength = 50 THEN
    INSERT INTO unlocks (user_a_id, user_b_id, unlock_type)
    VALUES (p_user_a, p_user_b, 'private_space') ON CONFLICT DO NOTHING;
  END IF;
END;
```

# 9. 보안 고려사항

- **편지 내용 봉인**: `content` 컬럼은 sender / recipient만 접근, RLS 강제
- **경로 비공개**: `visited_users` 배열 클라이언트 노출 금지, 서버에서만 사용
- **전달 권한 검증**: `current_holder === actor_user_id` 서버에서 반드시 검증
- **대화 기록 없음**: `visit_chats` 방문 종료 시 CASCADE 삭제
- **중앙우체국 조작 방지**: 월 제한 및 extra_count 검증 서버에서만 처리
- **태그 인젝션 방지**: 태그 입력 시 허용 문자 및 길이 제한 서버에서 검증
