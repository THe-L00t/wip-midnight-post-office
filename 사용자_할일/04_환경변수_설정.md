# 환경변수 설정 가이드

> 모든 키와 시크릿은 이 방식으로 관리합니다.
> `.env` 파일은 절대 GitHub에 올리지 마세요 (.gitignore에 포함되어 있습니다).

---

## Flutter 앱용 .env 파일

Flutter 프로젝트 루트에 `.env` 파일을 직접 생성하고 아래 내용을 채워넣으세요.

```env
# Supabase
SUPABASE_URL=https://여기에입력.supabase.co
SUPABASE_ANON_KEY=여기에입력

# 앱 설정
APP_BUNDLE_ID=com.nc.midnightpostoffice
```

> `SUPABASE_SERVICE_ROLE_KEY`는 클라이언트 앱에 포함하지 않습니다.
> Edge Function 환경변수로만 사용합니다.

---

## Supabase Edge Function 환경변수

터미널에서 아래 명령어를 순서대로 실행하세요.

```bash
# Firebase 서비스 계정 JSON (02_Firebase_설정.md Step 7에서 다운로드한 파일)
supabase secrets set FIREBASE_SERVICE_ACCOUNT='{ "type": "service_account", ... }'

# 앱 번들 ID (APNs 발송 시 필요)
supabase secrets set APP_BUNDLE_ID=com.nc.midnightpostoffice
```

등록된 환경변수 확인:
```bash
supabase secrets list
```

---

## 환경변수 목록 전체

| 변수명 | 사용 위치 | 출처 |
|--------|---------|------|
| `SUPABASE_URL` | Flutter 앱 | Supabase → Settings → API |
| `SUPABASE_ANON_KEY` | Flutter 앱 | Supabase → Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | Edge Function (자동 주입) | Supabase 자동 제공 |
| `FIREBASE_SERVICE_ACCOUNT` | Edge Function | Firebase → 서비스 계정 JSON |
| `APP_BUNDLE_ID` | Edge Function | `com.nc.midnightpostoffice` |

---

## 주의사항

- `.env` 파일은 `.gitignore`에 반드시 포함 (AI가 자동 추가)
- Firebase 서비스 계정 JSON 파일도 `.gitignore`에 포함
- `service_role` 키는 서버에서만 사용, 앱 코드에 절대 포함 금지
- 키가 실수로 GitHub에 올라갔다면 즉시 Supabase / Firebase에서 키 재발급
